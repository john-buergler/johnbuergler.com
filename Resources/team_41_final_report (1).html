I. Executive Summary:¶

We have built a Steam game recommendation engine, which will provide a handful of games it thinks a user will like based on their library information, including games they own as well as playtime in those games. The engine looks at the tags/genres of all Steam games, as well as their ratings, and attempts to find games with the closest genres and tags to a user's library. Then, it suggests the similar games that are also highly rated. We estimated the quality of our results by surveying various individuals on if they thought they would enjoy their recommended games, and determined that the model is successful in providing relevant recommendations.

II. Introduction:¶

There always comes a time where you really enjoy a piece of media, whether it be a book, movie, or video game, and you're left wanting more once it's over. When it comes to games, you might want to find something in the same genre, look for games made by the same developers, or experience the same themes or aesthetics. Steam is a platform with over 50,000 games and used by many PC players to find and play said games. One problem players may face is finding something similar to games they already enjoy based on multiple factors, sometimes even being unsure of what they're looking for. This project seeks to recommend games a player would be interested in based on games they have previously played.

We do this first by performing K-Means Clustering on all Steam games based on genre and tag data, then looking at a player's library and determining from which cluster to recommend games. By doing so, we are able to recommend them highly-rated games that match their preferences.

Cleaning Tags¶

While genres for each game are identified by the developers, tags are features that are posted by other users on a game's store page, making them community-based judgements of what a game is like. The API represents each tag on a game through the number of users that associated a game with said tag. This leads to the problem of some tags only being chosen by only a couple players and being potentially misleading. To better categorize each game, we selected the top ten most-chosen tags for each game performed one hot encoding on the these tag columns.

Pipeline Overview¶

We accomplish this data collection by using 9 functions:

  • get_games()
    • collects initial list of all steam games
  • get_game_data(int app_id)
    • given a game's app id, returns its metadata
  • get_all_game_data(dataframe df_game)
    • returns metadata from all games in the given dataframe
  • read_game_data_pkls()
    • converts pkl file and reads metadata to a singular dataframe
  • get_num_owners_of_game(dataframe df_game)
    • from given dataframe, gets the approximate number of owners for each game and appends a range of owners as a tuple for each
  • get_game_genres(dataframe df_game)
    • from given dataframe, cleans genre data by adding columns for each unique genre and appends to each a tuple if the game is of that genre or not (if true, append True)
  • get_game_tags(dataframe df_frame)
    • from given dataframe, cleans tags data by adding columns for each unique tag and appends to each a tuple if the game is of that tag or not (if true, append True)
  • get_user_game_data(int user_id)
    • given a user's Steam ID, gets the users data (games they own, friends, and game playtime)
  • def get_top_tags(df_tags, n):
    • given a dataframe of tags, collects the top 'n' tags and performs one hot encoding on them

We are collecting the metadata of multiple games and saving it in a PKL file because of the size of space it takes up. Then we convert the PKL file to be read into a dataframe. From there we clean up the dataframe. We do this by cleaning up the number of users who own each game and return as a range the number of users in string format, getting all unique game genres and performing one hot encoding to better format the data, and doing the same on the unique game tags.

def get_games():
    """ Uses SteamSpy API to retrieve list of all Steam Games
    
    Returns: df_games (pd.DataFrame): data frame containing all steam games
    """
    df_games = pd.DataFrame()

    for page in range(100):
        url = f'https://steamspy.com/api.php?request=all&page={page}'
        url_text = requests.get(url).text
    
        if url_text == '':
            print(f'{page} pages')
            break
        else:
            games = json.loads(url_text)
        
            for game in games.values():
                df_games = df_games.append(game, ignore_index=True)

    df_games = df_games.sort_values('appid').reset_index(drop=True)
    
    return df_games

import pandas as pd
import requests
import sys
import warnings

warnings.filterwarnings("ignore", category=FutureWarning) 

def get_game_data(app_id: int):
  """ Gets metadata about a given steam app.
    
    Args:
        app_id (int): The app ID for the game on Steam
    
    Returns:
        response_data (pd.Series): The API metadata for the games."""
  url = f"https://steamspy.com/api.php?request=appdetails&appid={app_id}"
  res = requests.get(url)
  if res.status_code != 200:
    print(res.status_code)
    sys.exit(0)
  res_json = res.json()
  response_data = pd.Series(res_json, dtype=object)
  return response_data

def get_all_game_data(df_games: pd.DataFrame):
  """Gets metadata about all steam apps.
    Args:
        df_games (pd.DataFrame): dataframe with all steam games

    Returns:
        df_game_data (pd.DataFrame): Dataframe with all game metadata, including tags and genres"""
  # batch requests by 1000 to reduce data loss in case of unexpected failure
  for num in range(53):
    df_game_data = pd.DataFrame(dtype=object)

    for i, app_id in enumerate(df_games['appid']):
      # to batch
      if i < 1000 * num:
        continue
      if i >= 1000 * (num+1):
        break

      print(i, app_id)
      try:
        # get game data and append to dataframe
        game_data = get_game_data(app_id)
        df_game_data = df_game_data.append(game_data, ignore_index=True)
      except Exception:
        print(i, app_id)
        raise

    # serialize dataframe and store on disk (needed to retain more complex data types) 
    df_game_data.to_pickle(f"./game_data{num}.pkl")
    print(f"wrote to ./game_data{num}.pkl")
    return df_game_data

import pandas as pd

def get_num_owners_of_game(df_game_data: pd.DataFrame):
    """ Cleans owner count data on a dataframe of steam game metadata.

    Args:
        df_game_data (pd.DataFrame): Dataframe with steam game metadata
    Returns:
        df_game_data (pd.DataFrame): same dataframe with cleaned up owner data"""
    # split the number of owners into low and high range values
    num_owners = df_game_data['owners'].map(lambda s: [int(num_str.replace(',', '')) for num_str in s.split(' .. ')])
    df_game_data['owners_range_low'] = num_owners.map(lambda p: p[0])
    df_game_data['owners_range_high'] = num_owners.map(lambda p: p[1])

    # remove old owners column
    del df_game_data['owners']

    return df_game_data

def get_game_genres(df_game_data: pd.DataFrame):
  """Cleans genre data on a dataframe of steam game metadata.

    Args:
        df_game_data (pd.DataFrame): Dataframe with steam game metadata
    Returns:
        df_game_data (pd.DataFrame): same dataframe with cleaned up genre data"""
  # get all possible genres
  genres = set()
  for genre_list in df_game_data['genre'].map(lambda s: s.split(', ')):
      for g in genre_list:
          if g != '':
              genres.add(g)

  # one hot encode genres
  for genre in genres:
      df_game_data[f'genre_{genre}'] = df_game_data['genre'].str.contains(genre)

  # remove old genre column
  del df_game_data['genre']

  return df_game_data

    Args:
        df_game_data (pd.DataFrame): Dataframe with steam game metadata

  # get all possible tags
  tags = set()
  for tag_list in df_game_data['tags'].map(lambda d: d.keys() if d else ()):
      for t in tag_list:
          if t != '':
              tags.add(t)

  # one hot encode tags
  for tag in tags:
      print(tag)
      tag_series = df_game_data['tags'].map(lambda d: d.get(tag, 0) if d else 0)

      df_game_data = df_game_data.copy()

  # remove old tags column
  del df_game_data['tags']

  return df_game_data

def get_top_tags(df_tags, n=10):
    """ returns dataframe where each game's top n tags have a value of True and
        all other tags have a value of False
        
    Args: 
        df_tags (pd.DataFrame): dataframe of tags
        n (int): top n tags, default is 10
        
    Returns: 
        df_top_tags (df): True for tags in the top n for that game, and false otherwise
    """
    # returns value for top tags, NaN for those not in the top
    top_tags = df_tags.apply(pd.Series.nlargest, axis=1, n=n)
    
    # replaces values and NaN with bool values
    df_top_tags = top_tags.applymap(lambda x: True if x > 0 else False)

    return df_top_tags

IV. Method:¶

ML Approach - Clustering¶

We chose to use a clustering algorithm since we want to output a collection of games most similar to what the user already plays. To do this, it is best to organize Steam's entire library into similar groups based on attributes that categorize different games. We can then find the group that contains the most similarity to a user's most-played games and recommend all the games in that group.

This is better than using a Classifer or Regression since clustering isn't supervised, meaning it automatically determines what games are most similar based on the given data and groups them accordingly rather than attempting to identify individual games that are closest to matching specific attributes.

We believe a user would enjoy a game they haven't played if it shares similar attributes to games they have already played. These attributes will include:

  • Genre
  • Tags

With this data, we will perform clustering on all Steam games available on the platform. Once the games have been grouped together based on their similarities made up of the attributes described above, we proceed to the final step.

We assume users prefer games that are highly rated by other players, thus we will also take into account user reviews of each game.

Our final assumption is that users prefer games that are similar to the most-played games in their library. This means that once the games are grouped together, we would recommend the highest-rated games from the cluster containing the largest amount of games and/or total playtime from the user's library, representing the games most similar to the user's preferences.

Before performing clustering, we also use Principle Component Analysis (PCA) for 3 reasons:

  • Reduces dimensionality (allowing us to plot in 2-dimensions with similar games close to each other)
  • Scale Normalization
  • Removes correlation between different features (preventing similar genres/tags distorting the clusters)

Potential pitfalls include the fact that we are only clustering with genres and tags, meaning several features from games are being ignored which could provide even more accurate clusters. Another problem is the fact that clustering is unsupervised, making the results and accuracy of the clusters and our final results hard to objectively prove.

# Contains prefixes for genres and tags
tag_prefix_tup = 'genre_', 'tag_'

def is_tag(feat, tag_prefix_tup=tag_prefix_tup):
    """ returns true if a given feature is a genre/tag
    
    Args:
        feat (str): feature being analyzed
        tag_prefix_tup (tuple(str)): list of recognized prefixes
        
    Returns:
        _ (bool): True/False depending on whether or not feature is a genre/tag
    """
    for tag_prefix in tag_prefix_tup:
        if feat.startswith(tag_prefix):
            return True
        
    return False

def strip_tag(feat, tag_prefix_tup=tag_prefix_tup):
    """ returns feature having removed recognized prefixes
    
    Args:
        feat (str): feature being analyzed
        tag_prefix_tup (tuple(str)): list of recognized prefixes
        
    Returns:
        feat (str): modified version of feat with removed prefixes
    """
    for tag_prefix in tag_prefix_tup:
        if feat.startswith(tag_prefix):
            feat = feat[len(tag_prefix):]
    
    return feat

def get_game_tag_tuple(game_row, tag_list=None):
    """ builds a list of all genres/tags which a game has
    
    Args:
        game_row (pd.Series): game to collect genres/tags from
        tag_list (list(str)): list of features from game recognized as genres/tags
        
    Returns:
        game_tag_list (tuple(str)): list of stripped genres/tags for the game
    """
    if tag_list is None:
        # re-compute
        tag_list = list(filter(is_tag, df_games_clean.columns))
    
    # collect all tags of a particular game
    game_tag_list = list()
    for feat in tag_list:
        if game_row[feat]:
            game_tag_list.append(strip_tag(feat))
            
    return tuple(game_tag_list)

VI. Discussion:¶

Model Performance and Shortcomings¶

We tested our recommendation engine with peers, and overall they were satisfied with the recommendations they were given. That being said, we have identified some shortcomings in our model and improvements we could make to future iterations of it:

  • To avoid recommending well-known games our users are already familiar with or very small indie games with specific niches, we initially limited recommendations to games within a certain range of reviews under 20,000 and above 500 reviews. Based on feedback from user testing, we made this upper/lower limit of reviews adjustable, as some users wanted to see more popular/niche games in their recommendations list.
  • Our model recommends games to users based on the cluster most represented in their top 50% of games by playtime. This model does not take into account games in other clusters the user enjoys playing, so the recommendations are limited and don’t reflect a user’s diversity in taste.
    • Since our model bases recommendations off of only one cluster, there are only as many different recommendation lists as there are clusters for a given upper limit of reviews. Testing this model on a number of users would result in repetitive recommendations.
    • In the future, we would adjust our model to take into account other clusters represented in a user’s play history as well as add a weighting system for play time when deciding how many games from each cluster to recommend.
  • When creating recommendations for a user, we look at playtime over the entire history of that user’s Steam profile. A user who has had a Steam account for several years may have had their tastes in video games change a lot in that time. Our model would then recommend games similar to something they enjoyed years ago, but may not enjoy now.
    • A future improvement could allow users to choose how far back in their play history we consider when making recommendations.

Ethical Considerations¶

Given that we are recommending video games, consequences of our project are relatively low-stakes compared to other applications of data science, such as financial modeling or health prediction. However, we have identified a few potential ethical considerations that may arise should usage of our project become widespread:

  • Our model doesn’t take into account age-appropriateness of games. There are tags for things such as nudity and 18+, but due to the sheer number of tags, these games will be included in clusters with games that don’t include these tags. Therefore, while a child may not have played any games with nudity, they could be recommended a game that does include it.
  • If a user plays a lot of games with loot boxes or a similar gambling component involved, the model will likely suggest games with similar mechanisms. This could inadvertently feed a user’s gambling addiction. It poses the ethical question: if a user already plays games that fuel an addiction, is it irresponsible for our model to recommend similar games?
